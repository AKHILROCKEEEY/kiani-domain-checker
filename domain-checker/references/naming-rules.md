# Name Research Rules

When brainstorming or evaluating a brand name, score every candidate against these criteria before checking domains.

## Hard Requirements (must pass all)

1. **ASCII-only.** The name must be writable using the 26 standard English letters (a-z) and optionally digits. No accents (ā, é, ñ), no special characters, no diacritics. If someone can't type it on a basic US keyboard, reject it.
2. **Single obvious spelling.** Say the name out loud. If the listener has to ask "how do you spell that?", it's too ambiguous. One pronunciation should lead to one spelling.
3. **Globally spellable.** A non-English speaker (Arabic, Farsi, Mandarin, Hindi, Japanese, Spanish) should be able to hear the name once and type it correctly into a browser. This means:
   - No silent letters (e.g., "knight" → they'll type "nite")
   - No English-only letter combos (e.g., "th", "ough", "wr") that other languages don't have
   - No ambiguous consonants (e.g., "c" could be /k/ or /s/ — prefer "k" or "s" explicitly)
   - No double letters that aren't pronounced (e.g., "ll" in "folkmeet" is fine, "ll" in "quill" where one is silent is not)
   - **Test:** Dictate the name to someone who speaks English as a second language. If they type it wrong, reject it.
4. **No negative homophones.** Read it fast, read it slow, read it tired. Does it sound like or look like a negative word in English? ("resent", "harm", "die", etc.) If yes, reject it.
5. **Cross-language safety.** Check the name doesn't mean something embarrassing, offensive, or negative in major European languages (French, German, Spanish, Italian, Portuguese, Dutch), Arabic, and any language common in the target market. Many brands have failed this check (e.g., "mist" means "manure" in German, "pet" means "fart" in French).

## Evaluation Criteria (score 1-5 each)

| Criterion | What to check |
|-----------|---------------|
| **Memorable** | Can someone hear it once and recall it an hour later? Short (2-3 syllables) wins. |
| **Pronounceable** | Works across accents — English, Arabic, Hindi, Russian, Mandarin speakers should all land on roughly the same pronunciation. |
| **Meaning** | Does the word mean something relevant? Does the meaning create a founder story? |
| **Distinct** | Google the name. Is it already a known brand, product, or common word that will bury you in search results? |
| **Feels right** | Subjective but real. Does it feel warm, human, communal? Or cold, corporate, techy? Match the product's soul. |

## Language Exploration

Draw inspiration from any language — English, Latin, Greek, Farsi, Japanese, Arabic, Sanskrit, Swahili — but the final brand name **must** be romanized into plain ASCII. Use the source language for the story, not the spelling.

**Process:**
1. Find a word with the right meaning in the source language
2. Romanize it into plain English letters
3. Check it passes the hard requirements above
4. Check it scores well on the evaluation criteria
5. Then check domain availability

**Example:**
- Source: Farsi همنوا (hamnavā) = "same melody"
- Romanized: `hamnava` — passes ASCII, pronounceable, meaningful
- Story: "It's Farsi for 'same melody' — because the best connections happen when people are on the same wavelength"

## Creative Name Tricks

When direct dictionary words are taken (they almost always are), use these techniques to generate available domains that still sound great.

### 1. Homophones & Double Meanings

Hide a second meaning inside a real word. The name reads as one thing but sounds like another — instant memorability and a built-in conversation starter.

| Technique | Example | What you see | What you hear |
|-----------|---------|-------------|---------------|
| Word collision | **meetball** | meet + ball | meatball |
| Hidden verb | **findling** | find + -ling | foundling |
| Noun as verb | **matchstick** | match + stick | matchstick (it sticks!) |
| Adjective swap | **folkward** | folk + ward | forward (to your folk) |
| Sound clash | **bondfire** | bond + fire | bonfire |
| Vowel wink | **peepal** | people (play) | peep + pal |

**Rules for homophones:**
- The "heard" word should be positive or playful — never negative
- Both readings (visual + auditory) should relate to the product
- Test it: say it in a noisy room. Does the listener smile or cringe?

### 2. Letter-Number Substitutions (Leetspeak-lite)

Replace letter sequences with numbers that sound the same. Keeps the name short and creates guaranteed-available domains.

| Letters | Number | Example | Reads as |
|---------|--------|---------|----------|
| -ate | 8 | **m8** | mate |
| -ight/-ite | 8 | **n8** | night → "n8.com" |
| -one/-won | 1 | **every1** | everyone |
| -to/-too/-two | 2 | **match2** | match too / match two |
| -for/-four | 4 | **4folk** | for folk |
| -ten | 10 | **of10** | often |
| -ine/-ein | 9 | **n9** | nine / nein |
| -ee | 3 | **fr3** | free |

**Rules for number substitutions:**
- Max ONE number substitution per name — `m8` is clever, `m8n8` is unreadable
- The number must be instantly obvious when read — no decoding required
- Avoid if the name will be spoken more than typed (radio ads, word-of-mouth)
- Works best for short, punchy names (2-4 characters + number)

### 3. Portmanteau & Syllable Blending

Fuse two words by overlapping shared sounds. The result feels like a real word.

| Word A | Word B | Overlap | Result |
|--------|--------|---------|--------|
| connect | echo | -ect/ech | **connecho** |
| belong | glow | -long/glow | **belonglow** |
| kin | innate | -in/in- | **kinnate** |
| thread | ready | -ead/read- | **thready** |
| folk | locale | -olk/loc- | **folkale** |
| match | alchemy | -atch/alch- | **matchemy** |

**Rules for portmanteaus:**
- The seam should be invisible — if you can see the stitching, it's forced
- Both source words should be recognizable within the blend
- Say it aloud: does it flow naturally or feel like a tongue-twister?

### 4. Prefix/Suffix Hacks

Add short affixes to a taken root word to create an available variant. These feel intentional, not desperate.

**Prefixes that work:**
| Prefix | Meaning | Example |
|--------|---------|---------|
| re- | again | **remeet**, **rebond** |
| un- | discover | **unfold** (as in unfold connections) |
| my- | personal | **myfolk**, **mymatch** |
| go- | action | **gomeet**, **gofind** |
| hey- | greeting | **heyfind**, **heyfolk** |

**Suffixes that work:**
| Suffix | Meaning | Example |
|--------|---------|---------|
| -ly | manner | **matchly**, **folkly** |
| -ish | approachable | **folkish**, **matchish** |
| -ful | abundance | **meetful**, **bondful** |
| -hub | center | **folkhub**, **bondhub** |
| -lab | experimental | **matchlab**, **folklab** |

### 5. Deliberate Misspelling

Swap, drop, or double a letter to make a taken word ownable. Only works if the misspelling is **obviously intentional**, not a typo.

| Technique | Example | Why it works |
|-----------|---------|-------------|
| Drop a vowel | **mtch** (match) | Minimal, techy |
| Swap vowel | **meetch** | Elongates the sound, playful |
| Double consonant | **folkk** | Visual emphasis |
| K for C | **konnekt** | Edgy, distinctive |
| Ph for F | **pholkmeet** | Too forced — DON'T do this |

**Rules for misspelling:**
- Must be immediately parseable — if someone squints, reject it
- The "correct" word should be obvious within 1 second
- Avoid if the product needs to project trust/professionalism (banks, health)
- Works best for casual, social, creative brands

### Combining Tricks

The best names often stack 2 techniques. Examples:
- **m8match** — number sub + repetition (mate + match)
- **rebondfire** — prefix + homophone (re-bond + bonfire)
- **folkl8** — root + number sub (folk + late → "folk late" gatherings)

**Limit: max 2 tricks per name.** More than that and it becomes a puzzle, not a brand.

## Scoring System

Every candidate **must** be scored before domain checking. Score each criterion 1-5. Calculate the **Total** (sum of all four). Higher-scoring names get checked first.

| Criterion | Score 1 | Score 5 |
|-----------|---------|---------|
| **Memorable** | Forgettable, generic, >3 syllables | Instantly sticky, 2 syllables, earworm quality |
| **Pronounceable** | Ambiguous vowels, consonant clusters, accent-dependent | One obvious pronunciation across all accents |
| **Distinct** | Common dictionary word, crowded search results | Zero Google noise, no existing brands |
| **Negative Check** | Sounds like or resembles a negative word in any major language | Clean across all checked languages |

**Priority rule:** Sort candidates by Total score descending. Check domains for the highest-scoring names first. Skip domain checks for any name scoring below 12/20 unless explicitly requested.

## Output Format

Names are split across **three files** by status:

| File | Contents | Table Format |
|------|----------|-------------|
| `tasks/domains-available.md` | WHOIS-confirmed available names | Full table with scores |
| `tasks/domains-taken.md` | Confirmed taken domains | Simplified (no scores) |
| `tasks/domains-rejected.md` | Failed brand review | Name, origin, meaning, rejection reason, date |

- Use `—` for scores/dates that don't exist
- Sort by Total score descending in available file, alphabetically in taken file
- When a domain is checked, add it to the correct file based on result
- Before checking new names, run `dedup-check.sh` to avoid re-checking
